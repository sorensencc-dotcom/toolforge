### 4. WSL2 find scope (item 1)

Add exclusions to the 3 identified `find` invocations:

- `claude-skills/scripts/convert.sh`
- `claude-skills/scripts/review-new-skills.sh`
- `rewrite-docs/scripts/cic-log-archival.sh`

Pattern: append `-not -path '*/node_modules/*' -not -path '*/.git/*'` to each `find` call that currently walks the full tree unscoped.

### 5. Script authoring conventions doc (items 2, 5 — policy only, no code change)

New file: `docs/meta/governance/script-authoring-conventions.md`

Contents:
- **Hook path resolution**: any new git hook installer must resolve paths dynamically — bash via `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`, Node via walk-up-to-`.git` (see `CIC-GOVERNANCE/scripts/setup-git-hook.mjs::findGitRoot` and `CIC-GOVERNANCE/scripts/governance-validate-precommit.sh` as reference implementations). Never hardcode a repo-relative path assuming standalone-clone context.
- **Migration dry-run**: any new backfill/migration script must accept a dry-run flag before it can write. See `utilities/roadmap-migration-helper.ps1` (`-DryRun`) as the reference pattern.

Add one reference line under "Governance & Design Standards" in `docs/meta/governance/global-operating-rules-cic-rewrite-labs.md` pointing to the new doc.
