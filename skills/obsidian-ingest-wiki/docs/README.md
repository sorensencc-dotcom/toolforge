## Overview

The `obsidian-ingest-wiki` skill orchestrates the Karpathy LLM-wiki pattern for Obsidian vault integration:

1. **Validates** staged raw sources against manifest (checks completeness, no corruption)
2. **Generates** a ready-to-copy prompt for Claude Code with paths substituted
3. **Supports** the 8-phase semantic ingest workflow (Ingest → Lint → Update → Cross-Ref → Lint → Log → Review → Commit)

The skill does not modify wiki content — that's Claude's job in an interactive session. The skill ensures staging is ready and provides context.

---

## Actions

### `validate`

Pre-flight check before synthesis. Verifies:
- Staging directory exists
- Manifest file present (`FILES.manifest.txt`)
- File count > 0
- Manifest is readable

**Returns**:
- Exit 0: Staging valid, ready for synthesis
- Exit 1: Staging invalid, synthesis blocked

**Example**:

```typescript
const result = await handler({ action: "validate" });

// Success:
// {
//   status: "valid",
//   stagingPath: "/vault/_kb-sync-staging/kb-sync/20260711-174821",
//   fileCount: 43,
//   errors: []
// }

// Error:
// {
//   status: "invalid",
//   errors: ["Manifest not found: ..."]
// }
```

### `prompt`

Generate Claude Code prompt with resolved paths. Includes:
- Resolved staging path (auto-discovered if not provided)
- Resolved vault root
- 8-phase workflow steps
- Schema reference pointer
- Constraints and expectations

**Example**:

```typescript
const result = await handler({ action: "prompt" });

// Output includes:
// === OBSIDIAN WIKI INGEST PROMPT ===
//
// **Staging Path:** /vault/_kb-sync-staging/kb-sync/20260711-174821
// **Vault Root:** /vault
// **Schema Document:** docs/targets/obsidian.md
//
// **Workflow Phases:**
// 1. Ingest — Identify new entities and concepts...
// ...
```

## Configuration

### Load from File

The skill reads `configs/obsidian.yaml`:

```yaml
vault_root: C:\dev\kb-sync\obsidian\vault
staging_dir: _kb-sync-staging
wiki_dir: wiki
mapping_rules:
  kb-sync: kb-sync
  notebooklm: notebooklm
```

### Override via Input

```typescript
const result = await handler({
  action: "validate",
  vaultRoot: "/custom/vault/path",
});
```

---

## Auto-Discovery

If no staging path provided, the skill:

1. Scans `vault/_kb-sync-staging/<repo>/` directory
2. Finds all timestamped subdirs (format: `YYYYMMDD-HHMMSS`)
3. Sorts by timestamp descending
4. Uses most recent

Ensures you don't have to manually track staging paths.

## Workflow Integration

```
npm run kb:sync:obsidian
        ↓
Stages raw sources (creates timestamped dir)
        ↓
await handler({ action: "validate" })
        ↓
Manifest validated ✓
        ↓
await handler({ action: "prompt" })
        ↓
Prints Claude Code prompt
        ↓
Claude Code session (phases 1-8)
        ↓
Creates/updates wiki pages, Index.md, Log.md
        ↓
git commit
        ↓
Wiki updated, staged sources locked for audit trail
```

## Schema Reference

All wiki synthesis follows the schema in `docs/targets/obsidian.md`:

- **Entity pages**: Single concrete system/module/feature
- **Concept pages**: Cross-cutting patterns or architectural concepts
- **Index.md**: Per-domain catalog of entities and concepts
- **Log.md**: Append-only operation log

The prompt generated by this skill includes a reminder to read `docs/targets/obsidian.md`.

## Error Handling

### Staging Not Found

**Cause**: No staging directories with timestamp format exist.

**Fix**: Run `npm run kb:sync:obsidian` first to stage raw sources.

### Manifest Missing

**Cause**: Staging exists but `FILES.manifest.txt` is missing (corrupted).

**Fix**: Re-run `npm run kb:sync:obsidian` to re-stage.

### Config Missing

**Cause**: `configs/obsidian.yaml` not found or `vault_root` not set.

**Fix**: Verify config path, set `OBSIDIAN_VAULT_ROOT` env var, or pass `vaultRoot` to handler.

## Constraints & Best Practices

1. **Raw sources immutable**: Do not edit files in `_kb-sync-staging/`. Only new stagings allowed.
2. **Wiki append-only**: Add/update pages; preserve history in Log.md.
3. **Manifest audit trail**: `FILES.manifest.txt` proves what was staged when.
4. **Human-in-loop synthesis**: No auto-updates to wiki content.
5. **Link to sources**: Every entity/concept page cites raw source(s) by absolute path.
