INTERACTIVE FEATURES
================================================================================

THEME TOGGLE
  JavaScript Function: toggleTheme()
  Storage: localStorage.kb-sync-theme
  Default: 'dark'
  Button Style: Fixed position (top-right, z-index: 1000)
  Label: "Light" (shows opposite of current theme)

  Implementation:
    1. Read current data-theme attribute
    2. Toggle between 'dark' and 'light'
    3. Update HTML attribute
    4. Update button label
    5. Save preference to localStorage
    6. CSS auto-applies via [data-theme="dark/light"] selector

CHART FILTERING
  Library: Chart.js v4.5.0 (CDN)
  Type: Bar chart
  Data:
    - Labels: 5 categories
    - Values: Broken link counts
    - Colors: Severity-based (red if 30+, orange if 15+, green otherwise)

  Interaction:
    - onClick handler on chart elements
    - activeFilter variable tracks selected category
    - Clicking same bar twice toggles filter off
    - Filter status displayed below chart with clear link
    - Table re-renders when filter changes

SEARCHABLE TABLE
  Library: Grid.js v5.0.2 (CDN)
  Columns: Source File | Target | Category | Severity | References
  Features:
    - Built-in search box (Grid.js native)
    - Pagination: 15 items per page
    - Sortable: Click column headers to sort
    - Styled with CIC theme colors

  Data Binding:
    - JavaScript array: brokenLinks (108 items)
    - Filtered by activeFilter if set
    - Mapped to table format (5-column array)
    - Grid.js renders and handles UX

RECOMMENDATIONS PANEL
  Generation Logic:
    1. Group broken links by category (5 categories)
    2. Count references per category
    3. Sort by priority (HIGH > MEDIUM > LOW)
    4. Generate actionable command for each

  Content per Recommendation:
    - Category name (e.g., "Missing Roadmaps")
    - Priority badge (colored)
    - Description with count (e.g., "16 unique, 16 references")
    - Copy-friendly shell command
    - Reference count display

  Copy Button:
    - Inline for each command
    - Uses navigator.clipboard.writeText()
    - Shows alert on success (can be improved with toast)
    - Styled to match accent color

SEARCH & FILTER COORDINATION
  Search: Grid.js native search across all columns
  Filter: activeFilter category from chart click
  Behavior:
    - Filter happens first (reduces visible rows)
    - Search applies within filtered results
    - Both are composable (AND logic)
    - Clear filter link resets to full view

================================================================================
DATA STRUCTURE
================================================================================

BROKEN LINKS ARRAY (JavaScript embedded)
  Sample structure:
  {
    "source": "reference/architecture.md",
    "target": "../roadmaps/unified-roadmap.md",
    "type": "broken_reference",
    "referenceCount": 6,
    "severity": "HIGH"
  }

  Processing:
    1. Links parsed from sync-report.json
    2. Reference count calculated (how many files link to same target)
    3. Severity assigned based on count:
       - 5+ = HIGH
       - 2-4 = MEDIUM
       - 1 = LOW
    4. Sorted by reference count (descending)

CHART DATA ARRAY (JavaScript embedded)
  Sample structure:
  {
    "category": "Missing Roadmap",
    "count": 16
  }

  Categories and Logic:
    - "Missing Roadmap": target includes "roadmap"
    - "Missing Operations": target includes "operation", "running", "monitoring", "sealing"
    - "Missing Architecture": target includes "architecture", "overview", "routing", "drift"
    - "External Reference": target starts with "../../" or "../" + code file extensions
    - "Template Variable": target includes "$", "template", "..."
    - "Other": everything else

RECOMMENDATIONS ARRAY (JavaScript embedded)
  Sample structure:
  {
    "category": "Missing Roadmaps",
    "priority": "HIGH",
    "description": "Create missing roadmap files (2 unique, 16 references)",
    "command": "mkdir -p docs/roadmaps && touch docs/roadmaps/unified-roadmap.md docs/roadmaps/cic-roadmap.md",
    "affectedCount": 16
  }

  Priorities: HIGH > MEDIUM > LOW
  Each has specific actionable command with copy button
