CATEGORIZATION LOGIC
=====================

Broken links are categorized using pattern matching:

1. Missing Roadmap
   - Pattern: target.includes('roadmap')
   - Count: 35 links
   - Example: ../roadmaps/unified-roadmap.md

2. Missing Operations
   - Pattern: target.includes(['operation', 'running', 'monitoring', 'sealing'])
   - Count: 18 links
   - Example: ../operations/monitoring.md

3. Missing Architecture
   - Pattern: target.includes(['architecture', 'overview', 'routing', 'drift'])
   - Count: 15 links
   - Example: ../architecture/routing.md

4. External Reference
   - Pattern: target.startsWith('../../') OR (target.startsWith('../') AND includes .ts/.json)
   - Count: 28 links
   - Example: ../../src/maal/router/maal-router-types.ts

5. Template Variable
   - Pattern: target.includes(['$', 'template'])
   - Count: 12 links
   - Example: rl-ref/\" + $_.name.replace('.md','') + \"

6. Other
   - All remaining unmapped patterns
   - Count: 0 (all categorized)

IMPACT SCORING ALGORITHM
=========================

Reference Count Calculation:
  1. Count occurrences of each unique target across all broken links
  2. Assign severity based on count:
     - HIGH: refCount >= 5 (red #E63946)
     - MEDIUM: 2 <= refCount < 5 (orange #F77F00)
     - LOW: refCount == 1 (green #06A77D)

Example:
  Target "../roadmaps/unified-roadmap.md" appears in 5 source files
  -> refCount = 5
  -> severity = HIGH (red)
  -> sorts to top of table
